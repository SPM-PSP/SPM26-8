# DDL-Master

> 基于多模态交互的智能个人日程助理

DDL-Master 是一个面向校园与轻办公场景的个人任务管理系统。
它聚焦“**碎片化信息难整理、任务优先级难判断、提醒容易遗漏**”这三个核心问题，提供从 **任务捕获 → 结构化解析 → 优先级排序 → 多通道提醒** 的完整闭环。

---

## 项目简介

在日常学习和工作中，任务往往散落在微信群通知、课程公告、邮件、语音记录等不同渠道中。传统日程工具通常要求用户手动整理和录入，导致：

* 任务录入成本高，容易遗漏
* 面对多个任务时缺少明确优先级
* 提醒方式单一，关键任务容易错过

DDL-Master 以微信小程序为主要载体，结合规则解析、语音识别和任务调度逻辑，帮助用户更轻松地完成任务管理。

---

## 核心功能

### 1. 多模态任务录入

支持多种输入方式，将碎片化信息快速转为结构化任务：

* **表单录入**：手动创建任务，填写标题、截止时间、重要度、备注等信息
* **剪贴板解析**：识别群通知、公告文本中的任务与时间信息
* **语音输入**：通过 ASR 将语音转写为文本，再进行任务解析

### 2. 任务管理与优先级调度

围绕“重要性 + 紧急度 + 截止时间”进行任务组织：

* 四象限任务看板
* 任务排序与临期任务高亮
* 任务状态流转：待办 / 进行中 / 已完成 / 逾期
* 支持长线任务拆分与阶段跟进

### 3. 智能展示与轻量交互

以微信小程序为主要界面，强调轻量、直观、快速操作：

* 首页快速查看今日任务
* 临期任务倒计时展示
* 任务详情与状态更新
* 个人中心与提醒设置

### 4. 分级提醒机制

通过不同提醒渠道增强任务触达效果：

* **微信订阅消息**
* **邮件提醒**
* **应用内弹窗/高亮提示**

> 当前版本以“提醒闭环可用、低成本可实现”为目标，短信等高成本能力仅保留扩展空间。

---

## 目标用户与典型场景

DDL-Master 主要服务以下几类场景：

* **高频碎片型用户**：每天接收大量群消息、通知和口头任务，容易漏记
* **单点爆发型用户**：在考试周、DDL周任务集中爆发，容易焦虑和拖延
* **长线规划型用户**：面对项目、论文、考证等长周期任务，难以持续推进
* **外部依赖型用户**：任务推进依赖他人反馈，容易中途遗忘跟进

## 系统流程

```text
任务输入 → 任务解析 → 任务管理 → 优先级排序 → 前端展示 → 提醒触达
```

## 技术栈


| 模块       | 技术方案                                  |
| ---------- | ----------------------------------------- |
| 前端       | 微信小程序原生开发（WXML / WXSS / JS）    |
| 后端       | Spring Boot / FastAPI                     |
| 数据库     | MySQL + Redis                             |
| AI与服务   | ASR 语音识别、LLM 解析接口、SMTP 邮件服务 |
| 协作与测试 | Git、Apifox / Postman、Docker             |

> 后端实现以模块拆分为主：核心业务、解析服务、提醒调度分别推进，并在阶段内统一集成。

## 开发计划

项目周期为 12 周，整体分为四个阶段：

1. **需求分析与设计**（第1\~2周）
   明确产品边界、模块划分、原型与接口规范
2. **开发阶段**（第3\~9周）
   分模块并行开发，完成基础框架、核心功能和系统集成
3. **测试与优化**（第10\~11周）
   进行单元测试、集成测试、真机兼容性测试与性能优化
4. **上线与维护**（第12周）
   准备审核材料、发布版本、整理答辩材料与项目文档

## 当前范围（MVP）

当前阶段优先实现以下最小可行闭环：

* 结构化任务录入
* 剪贴板文本解析
* 任务管理与四象限展示
* 基础优先级排序
* 微信/邮件提醒

暂不包含：

* 复杂多人协作
* 开放式自然语言任务理解
* 系统级跨平台日历同步
* 高成本短信提醒作为核心能力

## 项目亮点

* **多模态输入**：减少手动整理成本
* **规则 + AI 双层解析**：兼顾稳定性与灵活性
* **四象限 + 排序机制**：不仅记录任务，还辅助决策
* **分级提醒**：贴合小程序实际能力，形成可用闭环
* **低成本实现**：适合课程项目场景，具备较强落地性
