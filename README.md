# SPM26-8

# “DDL-Master” —— 基于多模态交互的智能个人日程助理

## 一、 项目背景

在快节奏的校园与职场生活中，用户的信息来源极度碎片化，如微信群、语音通知、邮件等，传统的“手动建日程”方式存在便捷和人性化的不足。
本项目旨在开发一款**具有“个人助理”属性**的日程管理系统。它不仅是被动的记录本，更是主动的规划师。
通过引入**​多模态输入​**与​**优先级调度算法**​，系统通过多种方式记录日程，并根据任务的重要紧急维度和时间维度进行优先级排序，最后通过多维方式（小程序微信提醒，邮件，短信）准时提醒用户，彻底根治“DDL 焦虑”。

## 二、 核心功能描述

DDL-Master 是一款基于多模态交互与智能调度算法的个人日程管理系统。系统技术架构严格遵循 “感知（Perception）— 决策（Decision）— 展现（Display）— 触发（Trigger）” 的闭环模型，旨在通过技术手段消除信息碎片化带来的认知负担。

### 模块 1：多模态智能录入交互

打破传统表单的枯燥限制，提供符合不同场景的三种录入方式：

1. ​**语音指令输入**​：系统调用 ASR（语音识别）转文字，并自动提取时间与事件。
2. ​**剪贴板/长文本智能解析**​：针对群里的长篇大论通知。系统借助 NLP（自然语言处理）或调用大模型 API，精准提取任务主体和精确的 DDL 时间点。
3. ​**结构化个人录入**​：保留传统表单，供用户进行任务细化（添加备注、子任务拆解、设置重要度标签）。

### 模块 2：智能优先级调度引擎

摒弃单一的列表，系统作为“助理”，需帮用户理清头绪：

1. ​**时间维度**​：按距离 DDL 的绝对倒计时排序，越近的任务越靠前，配合红黄绿三色视觉警报（紧急度极高的任务使用呼吸灯或醒目红色 UI）。
2. ​**重要程度维度**​：基于著名的“艾森豪威尔矩阵”，用户可标记任务的重要度。
3. ​**助理日报生成**​：系统首页“助理问候语”及今日核心任务摘要

### 模块 3：微信小程序

1. **移动端“快进快出”体验**：** 针对手机端碎片化场景，主打极速响应。首页即为当日任务看板，配合大按钮和滑动手势，确保用户在 5 秒内就能完成语音录入、剪贴板解析或任务勾划，真正做到“随手记、随时看”。
2. **缓解压力的沉浸式 UI**：** 针对“DDL 焦虑”，界面设计通过呼吸灯式柔和提醒、进度条丝滑填充动效，以及随任务紧急度变化的莫兰迪色系，从视觉心理学角度降低用户的操作门槛和心理负担。

### 模块 4：全方位触达与提醒机制

传统的 App 推送容易被系统屏蔽或被用户忽略，作为“个人助理”，必须保证 DDL 提醒的必达率：

1. ​**微信内置提醒**​：用户在使用小程序时，通过微信阅定实现消息提醒。
2. ​**电子邮件提醒**​：对于“重要但不紧急”的长期任务（如提前一周提醒），后端通过 SMTP 服务发送正式邮件，适合长周期项目管理。
3. ​**短信强提醒**​：​**终极防漏保险**​。对于极度重要（如考级报名缴费、期末大作业提交）且距 DDL 仅剩 2 小时仍未打卡标记完成的任务，系统调用云通信 API，向用户手机发送一条真实短信：“【DDL-Master】警告！您的任务[XXX]还有2小时到期，请尽快处理！”

## 三 产品目标和范围

| 用户行为分层                | 核心行为特征                                                                    | 核心痛点(Pain Points)                                                                                   | 细化方案 (FeatureSolutions)                                                                                                                                                       |
| ----------------------------- | --------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 单点爆发型 (The Sprinter)   | 任务平时处于休眠状态，在特定节点（如周五、考试周）呈爆炸式增长，对 DDL 极度敏感 | 拖延症，直到最后一刻才发现任务量超标，产生严重焦虑。                                                    | 【动态水位预警】：系统根据剩余时间与任务量的比例，自动计算“执行压强”。当压强超过阈值，小程序图标/首页强制切换为“战斗红色”，并自动置顶倒计时浮窗，营造物理紧迫感。             |
| 高频碎片型 (The Dispatcher) | 每天处理大量微信群通知、口头交代。任务本身不难，但多且杂，极易在切换中漏掉      | 记录成本高于执行成本。手动输入 10 条日程的时间足以完成其中 2 条，导致用户放弃记录。                     | 【静默收件箱 (Inbox)】：提供“粘贴即识别”的缓冲区。系统解析出任务后，不强制用户立即补全信息，而是将其放入“待确认槽位”。用户可在通勤等碎片时间一键滑动确认，实现“秒级录入”。  |
| 战略长线型 (The Marathoner) | 面对周期超过 1 个月的宏大目标。单日无明确 DDL，但需要持续投入                   | 容易陷入无反馈倦怠，由于没有即时 DDL 鞭策，任务始终停留在“重要不紧急”区，导致最后长线变短线，目标破产 | 【原子化拆解逻辑】：强制要求设置“里程碑节点”。系统将长线目标拆解为“周粒度”的小 DDL。若本周进度为 0，系统会自动提升该任务权重，将其从“计划做”强行挤入“今日必做”。          |
| 外部依赖型 (The Connector)  | 任务的开启或完成取决于他人（如：等老师回复、等同学发文档）                      | 阻塞性遗忘，任务卡在别人手里，自己就忘了跟进                                                            | 【Follow-up 追踪机制】：在任务创建时增加“等待他人”状态。系统不只提醒 DDL，更会在中间节点提醒用户“该去催一下某某了”，将被动等待转化为主动掌控。                                |
