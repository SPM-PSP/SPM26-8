# SPM26-8

# 项目名称：“DDL-Master” —— 基于多模态交互的智能个人日程助理

## 一、 项目背景与核心理念

在快节奏的校园与职场生活中，用户的信息来源极度碎片化（微信群、语音通知、邮件等），传统的“手动建日程”方式已成为负担。
本项目旨在开发一款**具有“个人助理”属性**的日程管理系统。它不仅是被动的记录本，更是主动的规划师。通过引入**​多模态输入（语音/文本解析）​**与​**科学的优先级调度算法**​，系统能像真实的助理一样听懂用户的任务指令，并根据任务的“重要紧急程度”自动排序，最后通过多维度的触达方式（弹窗/邮件/短信）准时提醒用户，彻底根治“DDL 焦虑”。


## 二、 核心功能描述

系统核心分为四大模块，全面覆盖任务的“输入 -> 分析 -> 展现 -> 触达”全生命周期：

### 模块 1：多模态智能录入交互（人机交互体验升级）

打破传统表单的枯燥限制，提供符合不同场景的三种录入方式：

1. ​**语音指令输入（Voice Input）**​：用户走在路上时，可按住按钮说话：“帮我记一下，明天下午三点要在主教开组会”。系统调用 ASR（语音识别）转文字，并自动提取时间与事件。
2. ​**剪贴板/长文本智能解析（Clipboard Parsing）**​：针对群里的长篇大论通知。用户复制文字后打开应用，系统自动读取剪贴板并弹窗提示：“检测到一条任务通知，是否生成日程？”系统借助 NLP（自然语言处理）或调用大模型 API，精准提取任务主体和精确的 DDL 时间点。
3. ​**结构化个人录入（Manual Input）**​：保留传统表单，供用户进行任务细化（添加备注、子任务拆解、设置重要度标签）。

### 模块 2：智能优先级调度引擎（核心业务逻辑）

摒弃单一的列表，系统作为“助理”，需帮用户理清头绪：

1. ​**时间维度（Timeline 排序）**​：按距离 DDL 的绝对倒计时排序，越近的任务越靠前，配合红黄绿三色视觉警报（紧急度极高的任务使用呼吸灯或醒目红色 UI）。
2. ​**重要程度维度（四象限法排序）**​：基于著名的“艾森豪威尔矩阵”，用户可标记任务的重要度。系统生成四象限看板：
   * ​*重要且紧急*​（马上做，如：今晚12点的期末大作业）
   * ​*紧急但不重要*​（授权做/快速做，如：下楼拿个快递）
   * ​*重要但不紧急*​（计划做，如：考研复习/背单词）
   * ​*不重要不紧急*​（少做/不做，如：刷短视频）
3. ​**助理日报生成**​：每天早上 8 点，系统首页生成一句“助理问候语”及今日核心任务摘要：“早上好，您今天有 2 个【重要且紧急】的任务即将到期，请优先处理。”

### 模块 3：多平台展现体系（前端架构选型）

考虑到大学生高频使用的场景，提供轻重搭配的展现形态：

1. ​**微信小程序（主阵地，推荐选型）**​：​**即用即走，免安装**​。极其适合剪贴板解析、语音录入等快速操作，且容易在班级群内部分享和推广。
2. ​**独立 App（进阶版/后续演进）**​：通过 Uni-app 等跨平台框架开发。App 端能实现更深度的系统权限获取（如常驻后台、系统级桌面小组件 Widget），提供更沉浸的“专注模式”（番茄钟）。

### 模块 4：全方位触达与提醒机制（通知矩阵）

传统的 App 推送容易被系统屏蔽或被用户忽略，作为“个人助理”，必须保证 DDL 提醒的必达率：

1. ​**应用内弹窗与红点（App-in Pop-up）**​：用户在使用小程序/App时，通过 WebSocket 或轮询实现实时任务即将到期的霸屏弹窗警告。
2. ​**电子邮件提醒（Email Alert）**​：对于“重要但不紧急”的长期任务（如提前一周提醒），后端通过 SMTP 服务发送正式邮件，适合长周期项目管理。
3. ​**短信强提醒（SMS Blast）**​：​**终极防漏保险**​。对于极度重要（如考级报名缴费、期末大作业提交）且距 DDL 仅剩 2 小时仍未打卡标记完成的任务，系统调用云通信 API，向用户手机发送一条真实短信：“【DDL-Master】警告！您的任务[XXX]还有2小时到期，请尽快处理！”
